# Backup/Restore Feature Implementation Plan

## Overview
Implement a robust JSON-based backup and restore system with proper validation, duplicate detection, and transactional updates. Move UI from toolbar buttons to list items on More page.

## Phase 1: Core Types & Utilities

### 1.1 Backup JSON Schema (`src/types/backup.ts`)
```typescript
interface BackupSchema {
  header: {
    schemaVersion: string;           // e.g., "1.0.0"
    appVersion: string;              // e.g., "2.6.47"
    createdAt: string;               // ISO timestamp
    checksum: string;                  // SHA-256 of data payload
  };
  data: {
    sessions: Session[];
    employeeName: string;
    email: string;
    jobTitle: string;
    department: string;
    onboardingCompleted: boolean;
    onboardingProgress: object;
    appSettings: object;
  };
}
```

### 1.2 Backup Utilities (`src/utils/backup.ts`)
- `createBackup(appData): Promise<BackupSchema>` - Create backup with checksum
- `calculateChecksum(data): string` - SHA-256 hash (using expo-crypto or web SubtleCrypto)
- `saveBackupToFile(backup)` - Trigger download (web) or save (mobile)
- `saveBackupToStorage(backup)` - Save to localStorage/AsyncStorage
- `loadBackupFromFile(): Promise<BackupSchema>` - Pick JSON file
- `loadBackupFromStorage(): Promise<BackupSchema | null>` - Load from storage
- `validateBackup(backup): ValidationResult` - Schema version + checksum validation
- `computeRecordChecksum(record): string` - For duplicate detection

## Phase 2: AppContext Enhancements

### 2.1 New Context Methods
- `restoreBackup(backup, options): Promise<RestoreResult>` - Deep merge/restore with transactional safety
- `getBackupSummary(): BackupSummary` - Quick stats for UI display
- Add `restoreInProgress` state for progress feedback

### 2.2 Restore Logic
```
1. Validate schema version (backward compat check)
2. Verify checksum
3. Parse and categorize records:
   - New (not in existing data)
   - Duplicate (identical by ID + checksum)
   - Conflicting (same ID, different content)
4. Create temp storage for transactional update
5. Apply changes atomically on success
6. Return detailed restore result
```

## Key Implementation Notes

### Checksum Calculation
- Compute SHA-256 of the minified JSON data (without header)
- Store in `header.checksum`
- On import, verify before processing

### Duplicate Detection
- For each session: `checksum = SHA256(sessionId + checkInTime + checkOutTime + reason)`
- Skip if identical checksum exists in current data
- Report as "already exists" count

### Transactional Updates
```
const tempBackup = { ...appData };
try {
  await mergeData(tempBackup, importData);
  setAppData(tempBackup);
  await saveData();
} catch {
  // Original appData unchanged
  Alert("Restore failed, no changes made");
}
```

### Progress Indicators
- For backups: Show "Processing X sessions..." with count
- For imports: Show "Validating... Importing X/Y sessions..."

### Edge Cases
- Backward compat: Check `schemaVersion` and provide defaults for missing fields
- Corrupted file: Catch JSON parse errors, show clear error
- Large files: Use chunked processing with `setTimeout` yields
- Empty backup: Show "No data to restore" message
